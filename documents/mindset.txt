LOGGER SERVICE CODE
─────────────────────────────────────────────────────────────────────────────

backend/services/loggerService.js:

const winston = require('winston');
const path = require('path');

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.combine(
    winston.format.timestamp({ format: 'YYYY-MM-DD HH:mm:ss' }),
    winston.format.json()
  ),
  transports: [
    new winston.transports.File({ 
      filename: path.join(__dirname, '../logs/error.log'), 
      level: 'error' 
    }),
    new winston.transports.File({ 
      filename: path.join(__dirname, '../logs/combined.log') 
    }),
    new winston.transports.Console({
      format: winston.format.simple()
    })
  ]
});

module.exports = logger;

SECURITY & BEST PRACTICES
─────────────────────────────────────────────────────────────────────────────

🔒 SECURITY MEASURES:

  1. Admin Authentication Required
     ├─ All backup endpoints require valid JWT token
     └─ Only admin can create, restore, or delete backups

  2. File Storage Security
     ├─ Backups stored outside public directory
     ├─ Added to .gitignore (never commit backups)
     └─ Optional: Encrypt backups for sensitive data

  3. Access Logging
     ├─ All backup operations logged
     ├─ Track who created/restored/deleted backups
     └─ IP address and timestamp recorded

  4. Backup Retention
     ├─ Auto-delete backups older than 30 days
     ├─ Configurable retention period
     └─ Prevents disk space issues

  5. Cloud Storage (Recommended)
     ├─ Upload backups to AWS S3, Azure Blob, etc.
     ├─ Off-site disaster recovery
     └─ Redundancy and availability


⚡ BEST PRACTICES:

  ✅ Schedule backups during low-traffic hours (2:00 AM)
  ✅ Test restore process regularly
  ✅ Monitor backup file sizes
  ✅ Keep multiple backup versions
  ✅ Store backups in different locations
  ✅ Compress backups to save space
  ✅ Document restore procedures
  ✅ Set up alerts for backup failures
